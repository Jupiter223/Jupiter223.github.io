出现 Invalid bound statement (not found) 异常

首先就是mp这个常出现的异常，我知道肯定是扫描路径啊映射之类的没有正确。一开始是在网上找方法，最后还是按照官网一条条对下来解决了。

  • 检查是不是引入 jar 冲突

  • 检查 Mapper.java 的扫描路径

    • 方法一：在 Configuration 类上使用注解 MapperScan
    • 方法二：在Configuration类里面，配置MapperScannerConfigurer（查看示例 (opens new window)）

  • 检查是否指定了主键？如未指定，则会导致 selectById 相关 ID 无法操作，请用注解 @TableId 注解表 ID 主键。当然 @TableId 注解可以没有！但是你的主键必须叫 id（忽略大小写）

  • SqlSessionFactory不要使用原生的，请使用MybatisSqlSessionFactory

  • 检查是否自定义了SqlInjector，是否复写了getMethodList()方法，该方法里是否注入了你需要的方法(可参考DefaultSqlInjector)

  • IDEA 默认的 build 步骤可能会引起 mapper 文件无法正常编译到对应的 resources 文件夹中，请检查 build 之后相关资源文件夹是否有对应的 xml 文件，如果没有，请调整 IDEA 的 build 设置，推荐调整为 Maven 或 Gradle 的 build

Mybatis动态SQL boolean值问题

在鸟列表页写一个动态SQL查询 其中有两个字段isDead和isOut分别用来查询生存状况和是否还在场地内,在mySQL里的类型是tinyint,默认值是0表示false,1表示true

用来传值的vo里一开始也是设置数据类型为boolean

然后写完动态sql以后 发现当选值为false时查询无效

去掉and isDead！=’’,可以正确的查出,但是这个查询条件就会一直在

于是我就将vo的数据类型改成Interger 传值设置为0或1

发现数值为0时也是失效的

所以这个问题就是

mybatis的if判断里面最好不要使用boolean值：

mybatis会默认把空值转为false。所以如果遇见前面传空值，这个字段在mybatis里面永远就是false了，可以使用数字类型代替，但是不要使用0作为参数。

如果使用1和2为参数可以正常查询,但是这样的话数据库的设置也要更改,从而会牵扯更多的地方.因此我并不想改数据库,就继续寻找解决方法.
